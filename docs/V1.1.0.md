# Magnot v1.1.0 implementation

## Adaptive selection architecture

`FerrousSelectionRuntime` is the only client event owner for tube selection, target highlighting, stored-region outlines, removal input, and preview rendering. It constructs exactly one `FerrousSelectionBackend` after checking `ModList`:

- `CreateFerrousSelectionBackend` is reflectively loaded only when Create is installed.
- `NativeFerrousSelectionBackend` is the self-contained fallback.

Both backends consume the same `FerrousRegion`, `FerrousSelectionView`, client region store, selection limits, and Magnot network payloads. They do not create Create glue entities or send Create placement/removal packets. Server-side `FerrousTubeItem`, `FerrousRegionSavedData`, ownership, durability, networking, Sable movement, and magnet rules remain authoritative.

### Create-backed reuse

The optional Create backend reuses:

- Create's glue raycast target helper;
- Catnip `Outliner` entries and natural refresh/fade behavior;
- Catnip face textures and Create-style line widths;
- Create glue placement sound events;
- Sable's Create outline transform extension when both mods are installed.

Magnot passes `assets/magnot/textures/entity/ferrous_region.png` into Catnip. It never uses or copies Create's glue texture.

Create's complete glue handler cannot safely be delegated to because it is hardcoded to `SuperGlueItem`, `SuperGlueEntity`, Create placement/removal packets, and private selection state. Magnot therefore reuses the safe presentation and targeting pieces while retaining the same Magnot interaction authority.

### Native fallback

The native backend reproduces the relevant glue-style flow with Minecraft APIs:

- first-corner feedback and second-corner preview;
- cancellation by sneaking;
- active target highlighting and stored-region outlines;
- passive and highlighted line widths;
- textured selection faces using Magnot's texture;
- vanilla slime sounds when Create is absent;
- transformed oriented boxes for Sable sublevels.

The native implementation is conceptually adapted from Create's MIT-licensed glue flow. Attribution is retained in `THIRD_PARTY_NOTICES.md`. No Create assets are present.

## Optional dependency safety

Direct Create imports exist only under `compat/create`. Common Magnot classes use interfaces and reflective factories after confirming the mod is loaded. Create, Ponder, Flywheel, and Registrate are compile-only unless `-Pwith_create=true` is supplied for a dev runtime.

Removed required dependencies:

- Create
- Ponder
- Flywheel
- Registrate
- Create Aeronautics dev runtime

Remaining optional dependencies:

- Create and its runtime libraries for the Create-backed selection path and crushing recipe
- Sable
- JEI
- EMI
- supported magnet/vacuum compatibility targets

## Adaptive recipe decisions

`AdaptiveMaterials` is the single implementation of material availability used by conditions, creative-tab visibility, JEI, and EMI.

External iron dust is available only when `c:dusts/iron` contains an item whose ID is not `magnot:iron_dust`.

| Material state | Active paste path | Magnot dust |
|---|---|---|
| External iron dust | `ferrous_paste_external_dust` with NeoForge difference ingredient | hidden and unobtainable |
| No external dust + Create | `create_crushing_iron_dust` and `ferrous_paste_create_dust` | visible and required |
| No external dust + no Create | `ferrous_paste_nugget_fallback` | hidden and unobtainable |

The external ingredient is `#c:dusts/iron` minus `magnot:iron_dust`; the fallback item cannot leak into the external recipe.

| Plate state | Active tube path |
|---|---|
| `c:plates/iron` populated | `ferrous_tube_plate` |
| `c:plates/iron` empty | `ferrous_tube_ingot_fallback` |

The custom `magnot:material_available` condition examines NeoForge's pending tag context during recipe reload. Inactive recipes are discarded before they reach the recipe manager.

## Recipe viewers

- JEI removes the dormant dust stack through `IIngredientManager.removeIngredientsAtRuntime`.
- EMI removes the dormant dust stack through `EmiRegistry.removeEmiStacks`.
- Viewer classes are isolated under `compat/jei` and `compat/emi` and compile against optional APIs only.
- Conditional recipes keep dormant dust out of recipe inputs, outputs, and usage searches.

## Performance changes

- Ferrous-region containment and segment tests now use allocation-free coordinate/slab math.
- Bulk magnet passes can reuse one source context and target-block candidate set.
- Simple Magnets and Item Collectors use lazy imperative filtering instead of unconditional stream/list allocation.
- SophisticatedCore reuses a pass-scoped query context.
- Sable candidate-sublevel discovery is cached per level, tick, and source/target block pair.

## Remaining risks

- Automated CI can verify compilation, dedicated-server startup, recipe conditions, and class isolation, but it cannot prove visual parity or input timing. Native/Create/Sable outline placement still requires an in-game client check before release.
- Create's glue handler internals remain version-sensitive. Magnot deliberately depends on the narrower public/de-facto raycast, Outliner, texture, and sound surfaces instead of Create's hardcoded glue state machine.
- Sable's transformed outline integration relies on Sable's current optional Create rendering extension when both mods are installed. The native path computes transformed corners independently.
